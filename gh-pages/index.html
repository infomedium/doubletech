    Publish Static Websites

    1. Prerequisites

    1.1. Python

    • If installed, skip next steps:

      python --version
# Python 3.10.11

    • From Microsoft store, install python (e.g. Python 3.10)

    • Open new terminal and check for python version. If not found, then add the following value to PATH environment variable. Refer Appendix #2.

      Env Variable Name Value
      PATH %USERPROFILE%\AppData\Local\Microsoft\WindowsApps

    • Repeat step for verification.

    1.2. Git

    • If installed, skip next steps:

      git --version
# git version 2.6.4.windows.1

    1.3. GitHub PAT

    • Follow instructions to generate a new Personal Access Token.
    • Save the PAT in a secure location.

    2. Server Setup

    2.1. Create a new repository

    • Repo Name: somesite
    • Visibility: Public
    • Add a README file
    • Add a .gitignore file with template as python

    2.2. Create a new branch

    • Branch Name: gh-pages

    2.3. Verify publish status

    • After branch is created, in a few minutes, the Deployments will show green tick.
    • Navigate to Settings > Pages
    • Click on site url at top of page to view published page.

    3. Client Setup

    3.1. Create Virtual Environment

    mkdir E:\ENV\VENV & cd E:\ENV\VENV
python -m venv .mkdocs-py-310

    Note

    A virtual environment in a common directory allows for reuse across multiple python projects.

    3.2. Install packages

    E:\ENV\VENV\.mkdocs-py-310\Scripts\activate.bat
python -m pip install --upgrade pip
pip install mkdocs-material

    3.3. Create Project Folder

    mkdir E:\GITHUB\somesite
cd E:\GITHUB\somesite

    3.4. Clone repo

    git clone https://github.com/someone/somesite.git --branch main main
git clone https://github.com/someone/somesite.git --branch gh-pages gh-pages

    3.5. Create new website

    cd main
mkdocs new . 
# INFO    -  Writing config file: .\mkdocs.yml
# INFO    -  Writing initial docs: .\docs\index.md

    3.6. Modify config

    site_name: My Docs
theme:
  name: material
site_dir: ../gh-pages

    3.7. Build website

    cd E:\GITHUB\somesite\main
mkdocs build

    3.8. Commit changes

    NOTE: Ensure email address is updated to proxy in each GH repo.
    Refer Appendix #4

    cd gh-pages
gid add *
git commit -a -m "my new changes"

    3.9. Publish website

    cd gh-pages
git push
# Username for 'https://github.com': someone
# Password for 'https://someone@github.com':

    3.10. Verify Changes

    • Launch website https://someone.github.io/somesite/ and verify changes manually.

    4. Create Workspace

    4.1. Project file

    • Save below as project.code-workspace file. E.g. E:\GITHUB\somesite\project.code-workspace

      {
    "folders": [
        {
            "path": "."
        }
    ],
    "settings": {
        "editor.formatOnSave": true,
        "files.exclude": {
            "**/.git": true,
            "**/.svn": true,
            "**/.vscode": true,
            "**/__pycache__": true,
            "**/.pytest_cache": true,
            "**/.venv": true
        },
    },
    "launch": {
        "configurations": [],
        "compounds": []
    }
}

    • Double click on workspace file to open VS Code.

    • Launch terminal within VS Code.

    5. APPENDIX

    1. How to check where python is installed?

    • Launch python from start menu icon.
    • Run below code:
    import sys
sys.executable
# 'C:\\Users\\XXX\\AppData\\Local\\Microsoft\\WindowsApps\\python.exe'

    2. How to add a new environment variable?

    Variable Name Value
    MY_KEY MY_VALUE

    Permanent:
    - In Windows Search, type Edit environment variables for your account . - In popup window, click on New... or Edit.. and update as required.

    Current Session Only:

    • To add a new name-value pair:
    SET MY_KEY=MY_VALUE
    • To append to existing value if variable name already exists:
    SET %MY_KEY%=%MY_VALUE%;%MY_KEY%

    3. How to check path of virtual environment?

    python -m pipenv --venv
# E:\ENV\PIPENV\main-6aBuroLU`

    4. How to update proxy email id before local commit?

    Ref: setting-your-commit-email-address

    The below step is important to avoid exposing real email address in GH repo to the public.

    cd "<folder containing .git folder>"
git config --local user.email "<private email-id from GH portal e.g. 123+someone@users.noreply.github.com"
git config --local user.name "<user name e.g. someone>"

    5. How to view git configuration changes?

    Run below command to see configurations at different level

    git config --local --list
# core.repositoryformatversion=0
# core.filemode=false

    git config --system --list
# core.symlinks=false
# core.autocrlf=true

    git config --global --list
# filter.lfs.required=true
# filter.lfs.clean=git-lfs clean -- %f
